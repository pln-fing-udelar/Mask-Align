Para generar los archivos test.32k.en, test.32k.es, answers.en y answers.es, que son usados por mask-align para alinear las respuestas se debe:

1) Ejecutar python .\preprocess_anotated.py --> esto genera test.en, test.es, answers-indexes.en.txt y answers.en.txt, answers-indexes.es.txt, answers.es.txt

2) Mover los archivos test.en y test.es al cluster

2) Ejecutar en el cluster:

       spm_encode --model=en.model --output_format=piece < test.en > test.32k.en
       spm_encode --model=es.model --output_format=piece < test.es > test.32k.es


3) Mover test.32k.en y test.32k.es de nuevo a la pc

4) Ejecutar:

     python process_answer_indexes.py
     python process_answer_indexes_es.py

5) Mover al cluster los archivos answers-indexes.32k.en.txt y answers-indexes.32k.es.txt

6) Ejecutar mask-align

7) - El archivo output.txt tiene los tokens con sus probabilidades y su clasificación correcta
   - El archivo output.txt2 tiene el string de la respuesta (tokenizada, hay q destokenizar con spm_decode)
   - El archivo output.txt3 tiene la respuesta marcada en la oración original con {{ y }} (tokenizado, hay que destokenizar con spm_decode)

8) Pasar el archivo output.txt3 destokenizado a la pc y ejecutar:
	python output-brackets-to-indexes

   Esto genera los índices de la respuesta y el archivo con las respuestas planas
